Struct and its memory allocation 2020-07-13

Overview

There is no object-oriented thought in golang, but the creators defined the Struct type to meet this need. In a Struct we can define some of fields which just like the attributes of the Class in other real object-oriented languages, but I recommend you’d better forget the thought of Class when using golang because the o-o programming is more flexible in golang. In a word, golang does not have real o-o programming but it has its own way to realize the thought of o-o.

Definition of fields

In Struct type we can define some of basic type such as string, int, pointer, slice, map and so on. Struct is the value type.
